# FlatWiki v0.8.1

Release-Datum: 21.02.2026

## Zusammenfassung

v0.8.1 ist ein Patch-Release mit Fokus auf Stabilität, Testbarkeit und robustere Konfigurationssicherheit.  
Nach dem v0.8.0-Feature-Release wurden vor allem Wartbarkeit und Betriebszuverlässigkeit verbessert.

## Highlights

- Versionsstand konsolidiert auf `0.8.0`/`0.8.1`-fähige Basis
  - `package.json` und Lockfile konsistent
- Test-Fundament mit Vitest eingeführt
  - neue Test-Skripte: `test`, `test:watch`, `test:coverage`
  - `vitest.config.ts` ergänzt
- Erste kritische Sicherheits-/Krypto-Tests implementiert
  - Passwort-Hashing/Verifikation (`scrypt`)
  - CSRF-Helper (Login-/Session-/Form-CSRF)
  - Wiki-Crypto-/Integritäts-Roundtrip (verschlüssertes Speichern + Integrität)
- Striktere Konfigurationsvalidierung (Fail-fast)
  - zentrale ENV-Validierung für Integer-/Boolean-/Enum-Werte
  - Pattern-Checks (z. B. Host/Scanner-Command)
  - 64-hex-Key-Validierung für kryptografische Schlüssel

## Sicherheit und Betrieb

- Frühzeitiges Stoppen bei ungültiger Konfiguration statt später Runtime-Fallbacks
- Verbesserte technische Absicherung durch automatisierbare Tests für sicherheitsrelevante Kernpfade

## Kompatibilität / Migration

- Keine beabsichtigten Breaking Changes im Anwendungsfluss
- Wichtig: ungültige ENV-Werte, die bisher stillschweigend toleriert wurden, können nun beim Start einen klaren Fehler auslösen (beabsichtigt)

## Empfohlene Checks nach Release

1. `docker compose up -d --build`
2. `npm run test` (oder im Container mit Node 20)
3. Login, Seiten speichern/bearbeiten/löschen, Backup/Restore prüfen
4. `config.env` auf gültige Werte und korrekte 64-hex-Keys prüfen
